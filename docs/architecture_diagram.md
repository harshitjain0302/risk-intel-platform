## Technology Stack

| Layer | Technology | Purpose |
|-------|-----------|---------|
| **Orchestration** | Apache Airflow | Workflow scheduling & monitoring |
| **Processing** | PySpark | Distributed ETL transformations |
| **Storage** | Parquet + PostgreSQL | Columnar files + relational warehouse |
| **ML Tracking** | MLflow | Experiment tracking & model registry |
| **Infrastructure** | Docker Compose | Local containerized deployment |

## Data Flow

1. **Ingestion**: Raw UNSW-NB15 CSV files (257K network flows)
2. **ETL**: Spark transformations with partitioning strategy
3. **Storage**: Parquet files (efficient columnar storage)
4. **Analytics**: PostgreSQL warehouse for SQL queries
5. **ML Training**: Classification models on attack detection
6. **Tracking**: MLflow logs parameters, metrics, artifacts

## Key Features

**Automated Pipelines**: End-to-end orchestration with Airflow  
**Scalable Processing**: PySpark for distributed transformations  
**Query-Ready Data**: SQL analytics on 257K+ security events  
**Experiment Tracking**: Reproducible ML workflows with MLflow  
**Partitioned Storage**: Efficient data organization (split + volume)  
**Production Patterns**: Idempotent loads, data validation, logging  

## Attack Categories Detected

- **Generic** (58K flows) - Automated scans
- **Exploits** (44K flows) - System vulnerabilities
- **Fuzzers** (24K flows) - Input validation attacks
- **DoS** (16K flows) - Denial of Service
- **Reconnaissance** (14K flows) - Network scanning
- **Analysis, Backdoor, Shellcode, Worms** (remaining categories)
